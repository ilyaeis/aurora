# Aurora — AI-Driven Open World 2D MMORPG

## The Idea

Aurora is an open-world 2D MMORPG where **everything is generated and driven by AI**. Instead of hand-crafting content, we use small open-source AI models to dynamically create the entire game world:

- **World Generation** — AI writes scenarios, quests, lore, and world events in real time
- **Character Creation** — NPCs with unique personalities, backstories, and dialogue generated by AI
- **Abilities & Skills** — Character abilities, spells, and skill trees designed by AI based on lore context
- **Dynamic Storytelling** — Branching narratives that adapt to player choices and server-wide events
- **Living Economy** — AI-managed markets, trade routes, and resource systems
- **Procedural Content** — Dungeons, biomes, items, and encounters that never repeat

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Frontend | React (TypeScript) |
| Backend | Python (FastAPI) |
| AI Engine | Small open-source models (e.g., Mistral, Phi, LLaMA) via Ollama or vLLM |
| Database | PostgreSQL + Redis |
| Real-time | WebSockets |
| Game Rendering | HTML5 Canvas / PixiJS |

## Philosophy

> Build less, generate more.

The goal is to prove that a small team (or even a solo developer) can create a content-rich MMORPG by offloading creative work to AI — making the game world feel vast, alive, and unpredictable without thousands of hours of manual content creation.
